- `firebase.json` includes:
  - `hosting` for the existing static export deployment.
  - `apphosting` for local-source App Hosting deployments to backend `ai-handiman`.
  - Firestore and Data Connect entries generated by Firebase init.
- `apphosting.yaml` sets Cloud Run sizing and references App Hosting secrets for Firebase web config values.
- Real Firebase web config values live in ignored `.env.local` for local development.
- Production Firebase web config values were uploaded as App Hosting secrets and are referenced by name in `apphosting.yaml`.
- Firebase Authentication is the selected identity/session engine, but Firebase Phone Auth SMS is not the selected OTP sender for real users. Real OTP SMS should be sent server-side through Africa's Talking, then exchanged for a Firebase custom token after verification.
- Future App Hosting secrets must include Africa's Talking credentials, Firebase Admin credentials, and a production `DATABASE_URL` when those integrations are implemented.
- Neon Free is the preferred study-phase hosted PostgreSQL target for Prisma. This keeps data off the local machine while preserving a standard Postgres interface.

## Important Deployment Rule

Do not set `output: "export"` in `next.config.ts` when deploying to Firebase App Hosting.

That setting is only for static export/classic Firebase Hosting. It caused the App Hosting Next adapter to fail in Cloud Build with:

```text
ENOENT: no such file or directory, open '/workspace/.next/standalone/.next/routes-manifest.json'
```

Root cause: App Hosting expects normal Next.js build output so it can create the server bundle and route manifest. Static export skips the standalone server output.

## Commands

Deploy App Hosting from local source:

```bash
firebase deploy --only apphosting:ai-handiman --project handimanautocare
```

Deploy classic static Hosting only:

```bash
corepack pnpm build
firebase deploy --only hosting --project handimanautocare
```

Use the classic static Hosting path only while the app remains static. Once the app needs server routes, sessions, SSR, or backend logic, App Hosting should be the primary deployment path.

## User-Friendly Error Guide

Error:

```text
Your project handimanautocare must be on the Blaze plan
```

Meaning: Firebase App Hosting cannot be enabled until billing is linked. Fix this in Firebase Console, then rerun backend creation or deployment.

Error:

```text
ENOENT: .next/standalone/.next/routes-manifest.json
```

Meaning: Next built a static export instead of App Hosting-compatible server output. Remove `output: "export"` from `next.config.ts`, rebuild, and redeploy App Hosting.

Error:

```text
permission-denied when reading or writing Firestore
```

Meaning: `firestore.rules` currently denies all access on purpose. Add collection-specific rules with tests before enabling Firestore-backed features.

Error:

```text
One or more rollouts failed
```

Meaning: Firebase accepted the deploy request, but Cloud Build failed later. Fetch the build logs with the build ID shown in the Firebase error:

```bash
gcloud builds log BUILD_ID --region us-east4 --project handimanautocare
```
